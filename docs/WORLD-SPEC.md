# 世界文档规范（WORLD-SPEC）

本文档定义了 AI GameStudio 世界文档的标准结构。世界文档是 Markdown 格式的文本，描述一个游戏世界的所有设定信息，供 LLM 作为 DM（Dungeon Master）时参考。

本文档同时作为 AI 生成世界文档时的系统提示。

## 设计目标

- **自包含**：一个世界文档应包含 DM 所需的全部信息
- **结构化**：使用标准 Markdown 章节组织，便于 LLM 解析
- **可扩展**：通过推荐章节适配不同类型的世界
- **可读性**：人类作者也能轻松阅读和编辑

## Frontmatter（可选）

世界文档可以包含 YAML frontmatter，声明默认启用的插件：

```yaml
---
plugins: [skill-check, combat, inventory, quest, faction, relationship, status-effect, codex]
---
```

Frontmatter 不是必需的，省略时使用项目默认插件配置。

## 必需章节

### `# 标题`

世界的名称，作为一级标题。应简洁有力，传达世界的核心特色。

### `## 世界背景`

整体设定描述，包括：
- 时代/纪元
- 核心概念（魔法、科技、武功等）
- 当前局势概述

建议 2-4 段文字。

### `## 地理`

世界中的重要地点。使用 `###` 子标题划分区域，每个地点用列表项描述：

```markdown
## 地理

### 区域名

- **地点A**：描述...
- **地点B**：描述...
```

### `## 势力/种族/门派`

世界中的重要阵营。推荐使用表格格式：

```markdown
| 名称 | 特点 | 所在地 |
|------|------|--------|
| 阵营A | 描述... | 地点 |
```

也可使用列表格式，视世界类型选择合适的章节标题（种族、势力、门派等）。

### `## 核心冲突`

世界中的主要矛盾线，使用列表格式：

```markdown
- **冲突A**：描述...
- **冲突B**：描述...
```

建议 3-5 个核心冲突，涵盖不同层面（政治、个人、超自然等）。

### `## DM 指引`

给 LLM 的叙事指导，包括：
- 叙事风格和基调
- 难度和规则倾向
- 特殊规则或限制

## 推荐章节

以下章节根据世界类型选择性添加。

### `## 魔法/科技/武功体系`

世界独有的能力系统。描述：
- 能力的来源和分类
- 使用规则和限制
- 等级或进阶体系

### `### 重要 NPC`

放在 `## DM 指引` 下方。定义可在游戏中出场的关键角色。每个 NPC 使用以下格式：

```markdown
#### 角色名（身份简述）
- 种族/外貌：...
- 性格：...
- 属性：{ key: value, ... }
- 随身物品：[...]
- 剧情作用：...
```

NPC 首次出场时，DM 应创建角色记录（具体格式由 core-blocks 插件定义）。

### `### 属性系统`

定义角色使用的属性名称和含义：

```markdown
### 属性系统
- **气血**（HP）：生命值
- **内力**（MP）：施展能力消耗
- **攻击**：物理伤害
```

### `### 状态变更规则`

定义何时 DM 应记录状态变更（具体 block 格式由 core-blocks 插件定义）：

```markdown
### 状态变更规则
当以下情况发生时，应记录状态变更：
- 玩家获得/失去物品
- 角色属性变化
- 到达新地点
```

### `### NPC 管理规则`

定义 NPC 的创建和更新时机（具体 block 格式由 core-blocks 插件定义）：

```markdown
### NPC 管理规则
- 首次出场时创建角色记录
- 状态变化时更新对应字段
- 属性只写变化部分
```

### `### 事件管理规则`

定义游戏事件的生命周期管理（具体 block 格式由 core-blocks 插件定义）：

```markdown
### 事件管理规则
- 接受任务 → 创建事件
- 重大进展 → 推进事件
- 任务完成 → 完结事件
- 任务失败 → 终止事件
```

### `### 玩法触发指引`

描述世界中各类游戏机制的自然触发时机。此章节告诉 DM **何时**触发机制，而非格式细节（格式由对应插件定义）：

```markdown
### 玩法触发指引

以下描述了本世界中各类游戏机制的自然触发时机：

- **技能检定**：[何时需要检定]
- **战斗**：[何时进入战斗]
- **物品管理**：[何时记录物品]
- **任务追踪**：[何时创建/推进任务]
- **阵营声望**：[何时影响声望]
- **NPC 关系**：[何时变化关系]
- **状态效果**：[何时施加/移除效果]
- **知识发现**：[何时记录新知识]
```

## 示例骨架

```markdown

## 地理

- **[地点1]**：[描述]
- **[地点2]**：[描述]

## [势力/种族/门派]
```

## AI 生成指引

当 AI 生成世界文档时，应遵循以下原则：

1. **严格按照上述结构** 组织章节
2. **内容要具体** — 地点要有名字和特色描述，NPC 要有性格和属性
3. **冲突要多层次** — 涵盖宏观（世界级）和微观（个人级）矛盾
4. **DM 指引要实用** — 给出明确的叙事风格指导和规则
5. **状态变更规则必须包含** — 列出触发状态变更的叙事条件，不要包含 block 格式细节
6. **玩法触发指引必须包含** — 用世界观语言描述各类游戏机制的自然触发时机
7. **NPC 至少 3-4 个** — 覆盖不同角色定位（情报、商人、盟友、对手）
8. **不要包含 json:xxx 格式细节** — block 格式由插件系统定义，世界文档只描述叙事级触发条件
9. **语言与请求一致** — 如果用户要求中文世界，全部内容用中文撰写
