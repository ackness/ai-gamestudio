# 世界文档规范（WORLD-SPEC）

本文档定义了 AI GameStudio 世界文档的标准结构。世界文档是 Markdown 格式的文本，描述一个游戏世界的所有设定信息，供 LLM 作为 DM（Dungeon Master）时参考。

本文档同时作为 AI 生成世界文档时的系统提示。

## 设计目标

- **自包含**：一个世界文档应包含 DM 所需的全部信息
- **结构化**：使用标准 Markdown 章节组织，便于 LLM 解析
- **可扩展**：通过推荐章节适配不同类型的世界
- **可读性**：人类作者也能轻松阅读和编辑

## Frontmatter（可选）

世界文档可以包含 YAML frontmatter，声明默认启用的插件：

```yaml
---
plugins: [choices, memory]
---
```

Frontmatter 不是必需的，省略时使用项目默认插件配置。

## 必需章节

### `# 标题`

世界的名称，作为一级标题。应简洁有力，传达世界的核心特色。

### `## 世界背景`

整体设定描述，包括：
- 时代/纪元
- 核心概念（魔法、科技、武功等）
- 当前局势概述

建议 2-4 段文字。

### `## 地理`

世界中的重要地点。使用 `###` 子标题划分区域，每个地点用列表项描述：

```markdown
## 地理

### 区域名

- **地点A**：描述...
- **地点B**：描述...
```

### `## 势力/种族/门派`

世界中的重要阵营。推荐使用表格格式：

```markdown
| 名称 | 特点 | 所在地 |
|------|------|--------|
| 阵营A | 描述... | 地点 |
```

也可使用列表格式，视世界类型选择合适的章节标题（种族、势力、门派等）。

### `## 核心冲突`

世界中的主要矛盾线，使用列表格式：

```markdown
- **冲突A**：描述...
- **冲突B**：描述...
```

建议 3-5 个核心冲突，涵盖不同层面（政治、个人、超自然等）。

### `## DM 指引`

给 LLM 的叙事指导，包括：
- 叙事风格和基调
- 难度和规则倾向
- 特殊规则或限制

## 推荐章节

以下章节根据世界类型选择性添加。

### `## 魔法/科技/武功体系`

世界独有的能力系统。描述：
- 能力的来源和分类
- 使用规则和限制
- 等级或进阶体系

### `### 重要 NPC`

放在 `## DM 指引` 下方。定义可在游戏中出场的关键角色。每个 NPC 使用以下格式：

```markdown
#### 角色名（身份简述）
- 种族/外貌：...
- 性格：...
- 属性：{ key: value, ... }
- 随身物品：[...]
- 剧情作用：...
```

NPC 首次出场时，DM 应通过 `json:state_update` 的 `characters` 字段创建角色记录。

### `### 属性系统`

定义角色使用的属性名称和含义：

```markdown
### 属性系统
- **气血**（HP）：生命值
- **内力**（MP）：施展能力消耗
- **攻击**：物理伤害
```

### `### 状态变更规则`

定义何时 DM 应输出 `json:state_update` 块：

```markdown
### 状态变更规则
当以下情况发生时，输出 `json:state_update`：
- 玩家获得/失去物品
- 角色属性变化
- 到达新地点
```

### `### NPC 管理规则`

定义 NPC 的创建和更新时机：

```markdown
### NPC 管理规则
- 首次出场时创建（省略 character_id，设 role: "npc"）
- 状态变化时更新（使用 character_id）
- 属性只写变化部分
```

### `### 事件管理规则`

定义游戏事件的生命周期管理：

```markdown
### 事件管理规则
- 接受任务 → json:event（action: "create"）
- 重大进展 → json:event（action: "evolve"）
- 任务完成 → json:event（action: "resolve"）
- 任务失败 → json:event（action: "end"）
```

## 示例骨架

以下是一个最小可用的世界文档模板：

```markdown
# [世界名称]

## 世界背景

[2-4 段描述世界的整体设定、时代背景、核心概念]

## 地理

### [主要区域]

- **[地点1]**：[描述]
- **[地点2]**：[描述]
- **[地点3]**：[描述]

## [势力/种族/门派]

| 名称 | 特点 | 所在地 |
|------|------|--------|
| [阵营1] | [描述] | [地点] |
| [阵营2] | [描述] | [地点] |

## 核心冲突

- **[冲突1]**：[描述]
- **[冲突2]**：[描述]
- **[冲突3]**：[描述]

## DM 指引

### 叙事风格
- [风格要点1]
- [风格要点2]

### 重要 NPC

#### [角色名]（[身份]）
- 外貌：[描述]
- 性格：[描述]
- 属性：[key: value]
- 随身物品：[列表]
- 剧情作用：[描述]

### 状态变更规则
当以下情况发生时，输出 `json:state_update`：
- [触发条件1]
- [触发条件2]

### NPC 管理规则
- 首次出场时通过 `json:state_update` 创建
- 状态变化时更新

### 事件管理规则
- 接受任务 → `json:event`（action: "create"）
- 任务完成 → `json:event`（action: "resolve"）
```

## AI 生成指引

当 AI 生成世界文档时，应遵循以下原则：

1. **严格按照上述结构** 组织章节
2. **内容要具体** — 地点要有名字和特色描述，NPC 要有性格和属性
3. **冲突要多层次** — 涵盖宏观（世界级）和微观（个人级）矛盾
4. **DM 指引要实用** — 给出明确的叙事风格指导和规则
5. **状态变更规则必须包含** — 确保游戏机制能正常运转
6. **NPC 至少 3-4 个** — 覆盖不同角色定位（情报、商人、盟友、对手）
7. **语言与请求一致** — 如果用户要求中文世界，全部内容用中文撰写
